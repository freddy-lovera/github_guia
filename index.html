📖 Vocabulario clave

GitHub — Una página web donde los programadores guardan sus proyectos. Es como Google Drive, pero especial para código.

Repositorio (repo) — La "carpeta" de tu proyecto en GitHub. Ahí dentro viven tus archivos HTML, CSS, imágenes, etc.

GitHub Pages — Un servicio GRATIS de GitHub que publica tu repositorio como una página web. Cualquiera puede verla con un link.

README.md — Un archivo donde escribes una descripción de tu proyecto. Se muestra automáticamente en la página principal de tu repo.

Commit — Es como presionar "Guardar" en un videojuego. Cada commit guarda los cambios que hiciste.

URL / Link — La dirección web (ej: https://tunombre.github.io/mi-pagina/) que compartes para que otros vean tu página.

00

Crear una cuenta en GitHub

Regístrate gratis. Sin esto no puedes usar GitHub Pages.

▾
¿Para qué sirve este paso? Necesitas una cuenta en GitHub para poder crear repositorios y publicar páginas. Es como crear una cuenta en Instagram o TikTok, pero para programar. Es totalmente gratis y no necesitas tarjeta de crédito.

🔽 Sigue estos pasos EN ORDEN:

1
Abre Google Chrome, Edge o Firefox (el navegador que uses normalmente).
💡 Si no sabes qué es un navegador: es el programa donde ves páginas web, el icono azul de "e" o el de colores.
2
En la barra de direcciones (arriba del todo), escribe: github.com y presiona Enter.
💡 Te aparecerá una página con un fondo oscuro y muchas letras.
3
Busca y haz clic en el botón «Sign up» que está en la esquina superior derecha.
💡 Sign up significa "Registrarse" en inglés.
4
Completa el formulario de registro:
📧 Email: tu correo electrónico (el que usas en el colegio o tu correo personal)
🔑 Password: una contraseña que recuerdes (mínimo 8 caracteres)
👤 Username: tu nombre de usuario (elige algo como tu-nombre o tu-apellido)
⚠️ El nombre de usuario debe ser ÚNICO. Si el que quieres ya lo usa otra persona, prueba con otra combinación.
5
GitHub te pedirá que resuelvas un puzzle de verificación (arrastrar una figura hasta su lugar). Esto es para confirmar que no eres un robot.
6
Te llegará un código de verificación al correo electrónico que registraste.
💡 Abre tu correo (Gmail, Outlook, etc.), busca el mensaje de GitHub y copia el código de 6 dígitos.
7
Escribe ese código en la página de GitHub y presiona Enter.
💡 Si no encuentras el correo, revisa la carpeta de SPAM o Correo no deseado.
🔍 Así debería verse: Una vez verificada tu cuenta, GitHub te mostrará una página de bienvenida con algunas preguntas (puedes saltarlas haciendo clic en "Skip personalization"). Luego verás tu dashboard (panel principal) de GitHub.
github.com - Panel principal (dashboard)
🎉 ¡Bienvenido a GitHub, tu-usuario!
Este es tu panel de control. Aquí verás tus repositorios.
💡Recomendación importante: Elige un nombre de usuario que puedas poner en tu currículum. Ejemplos buenos: ana-perez, carlos-martin, sofia-dev. Evita: xXpro_gamerXx, messi_10_fan.
⚠️Error común: Si te dice "Username already taken" significa que ese nombre ya lo usa otra persona. Prueba agregando un número o tu segundo apellido. Ej: ana-perez23.
✅¡Cuenta creada exitosamente! Tu perfil público es: github.com/tu-usuario
01

Crear un repositorio nuevo

El repositorio es la "carpeta" donde vivirán los archivos de tu página.

▾
¿Qué es un repositorio y por qué lo necesito? Imagina que GitHub es un edificio de departamentos. Tu repositorio es tu departamento: ahí guardas tus cosas (tus archivos HTML, CSS, imágenes). Sin repositorio, no puedes publicar nada.

🔽 Crea tu primer repositorio:

1
Abre github.com y asegúrate de haber iniciado sesión (tu foto de perfil debe aparecer arriba a la derecha).
2
Busca el botón verde «New» y haz clic en él. O una forma más fácil: escribe directamente en la barra del navegador: github.com/new y presiona Enter.
💡 El botón «New» está al lado izquierdo, arriba de la lista de repositorios, tiene un icono 📕.
3
Se abrirá un formulario. En el campo «Repository name» (nombre del repositorio) escribe el nombre de tu proyecto. Ejemplo:
mi-primera-pagina
⚠️Reglas del nombre: SIN espacios, SIN mayúsculas, SIN caracteres especiales (ñ, tildes). Usa guiones - para separar palabras.
4
Elige la visibilidad: Más abajo verás dos opciones: «Public» y «Private». Selecciona «Public».
💡 «Public» significa que cualquiera puede ver tu página. «Private» es solo para ti. Para GitHub Pages necesitas que sea Público.
5
Marca la casilla que dice «Add a README file». Esto crea un archivo de presentación para tu proyecto.
💡 README es como la "portada" de tu proyecto. Ahí puedes escribir de qué trata.
6
Haz clic en el botón verde «Create repository» que está al final del formulario.
💡 No toques ninguna otra opción. Solo nombre, público, README y crear.
🔍 Así debería verse: Después de hacer clic en «Create repository», GitHub te llevará a la página de tu repositorio. Arriba deberías ver algo como: tu-usuario/mi-primera-pagina. Abajo debe aparecer el archivo README.md que se creó automáticamente.
github.com/tu-usuario/mi-primera-pagina
⬆ Aquí dice: tu-usuario / mi-primera-pagina (nombre del repo)
Marca «Public» visible
📄 Archivo README.md ya creado (lo ves en la lista de archivos)
✅¡Repositorio creado! Ahora tienes tu carpeta en la nube. En los siguientes pasos vamos a llenarla con los archivos de tu página web.
02

Preparar tu archivo index.html

Tu página principal debe llamarse exactamente index.html.

▾
¿Por qué index.html? GitHub Pages (y todos los servidores web del mundo) buscan automáticamente un archivo llamado index.html para mostrar la página principal. Si tu archivo se llama pagina.html o inicio.html, NO funcionará. Debe llamarse EXACTAMENTE index.html, todo en minúsculas.

🔽 Crea tu archivo index.html:

1
Abre el Bloc de Notas (Windows) o cualquier editor de texto.
💡 En Windows: presiona la tecla Windows, escribe "Bloc de notas" y haz clic. También puedes usar VS Code si lo tienes instalado.
2
Copia y pega el siguiente código exactamente como está aquí abajo:
código para copiar
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0">
    <title>Mi primera página web</title>
</head>
<body>
    <h1>¡Hola, mundo!</h1>
    <p>Esta es mi primera página publicada en Internet.</p>
    <p>Hecha por [ESCRIBE TU NOMBRE AQUÍ] 🚀</p>
</body>
</html>
3
En el código, cambia [ESCRIBE TU NOMBRE AQUÍ] por tu nombre real.
💡 Por ejemplo: Hecha por Ana Pérez 🚀
4
Guarda el archivo: Presiona Ctrl + S (o ve a Archivo → Guardar como). En la ventana que aparece:
1️⃣ En "Tipo" selecciona "Todos los archivos (*.*)"
2️⃣ En "Nombre" escribe exactamente: index.html
3️⃣ Guarda en tu Escritorio para encontrarlo fácil después
⚠️ Es MUY importante que selecciones "Todos los archivos", porque si no, el Bloc de Notas lo guardará como index.html.txt y NO funcionará.
⚠️Error común #1: Si el archivo se ve como index.html.txt, lo guardaste mal. Repite el paso 4 asegurándote de seleccionar "Todos los archivos" en el tipo.
⚠️Error común #2: Si el archivo se llama Index.html (con I mayúscula) puede dar problemas. Asegúrate de que sea todo en minúsculas: index.html.
💾Archivo guardado correctamente. Ahora tienes un archivo index.html en tu Escritorio. Deberías ver un icono que se parece a un navegador web (Chrome o Edge). Si haces doble clic en él, se abrirá tu página en el navegador.
03

Subir tu index.html a GitHub

Subes el archivo desde tu computadora al repositorio en la nube.

▾
¿Qué vamos a hacer? Vas a tomar el archivo index.html que guardaste en tu Escritorio y lo vas a "subir" (copiar) a tu repositorio en GitHub. Así quedará guardado en Internet.

🔽 Sube tu archivo paso a paso:

1
Abre tu repositorio en GitHub. Deberías estar en una página que se ve como: github.com/tu-usuario/mi-primera-pagina
💡 Si no estás ahí, abre el navegador, ve a github.com y haz clic en tu repositorio «mi-primera-pagina».
2
Busca el botón «Add file» (Agregar archivo). Está en la parte de arriba, del lado derecho, al lado del botón verde «Code». Haz clic en él.
💡 Se abrirá un menú con dos opciones.
3
En el menú, haz clic en «Upload files» (Subir archivos).
💡 Upload files significa "Subir archivos" en inglés.
4
Arrastra tu archivo index.html desde tu Escritorio hasta el recuadro punteado que dice "Drag files here..." que aparece en el centro de la pantalla.
💡 También puedes hacer clic en "choose your files" y buscar el archivo en tu carpeta.
5
Verás que el archivo aparece listado con un icono 📄. Baja con el scroll hacia abajo.
6
En el campo «Commit changes» escribe un mensaje corto que describa lo que subiste. Ejemplo:
Subo mi primera página web
💡 "Commit" es como cuando le dices "Guardar" a un archivo. El mensaje te ayuda a recordar qué hiciste.
7
Haz clic en el botón verde «Commit changes» que está abajo del mensaje.
💡 No toques ninguna otra opción. Solo escribe el mensaje y presiona el botón verde.
🔍 Así debería verse: Después del clic, GitHub te llevará de vuelta a la página de tu repositorio. Ahora deberías ver dos archivos en la lista: README.md (el que se creó solo) y index.html (el que acabas de subir). ¡Felicidades, tu archivo ya está en la nube!
Repositorio con archivos
📄README.md
📄index.html ✅ Subido ahora
📦¿Tienes más archivos? Si tu página tiene CSS (estilo.css), imágenes o JavaScript, puedes subirlos todos juntos. En el paso 4 (paso 4 de arrastrar), selecciona todos tus archivos a la vez y arrastra el grupo completo.
04

Activar GitHub Pages

Le decimos a GitHub que convierta tu repositorio en una página web.

▾
¿Qué estamos haciendo? Tu repositorio actualmente solo guarda archivos. GitHub Pages es un "interruptor" que le dice a GitHub: "Oye, los archivos de este repositorio quiero que se vean como una página web". Lo activas una sola vez y funciona para siempre.

🔽 Activa GitHub Pages:

1
Dentro de tu repositorio, busca la pestaña «Settings» (Configuración) en la parte de arriba. Está al final del menú horizontal. Haz clic.
💡 El menú horizontal empieza con: Code, Issues, Pull Requests, ... y al final está Settings.
2
Mira el menú de la izquierda (es una lista vertical). Busca donde dice «Pages» y haz clic.
💡 Pages está en la sección "Code and automation", más o menos en la mitad de la lista. Tiene un icono 📘.
3
Verás una sección que dice «Branch» (Rama). Por defecto dice «None». Haz clic en el menú desplegable que dice «None» y selecciona «main».
💡 «main» es el nombre de la rama principal de tu repositorio. Ahí están tus archivos.
4
Al lado del menú de Branch hay otro menú que dice «/ (root)». Déjalo así, no lo cambies.
💡 «/ (root)» significa "la raíz", o sea, la carpeta principal. Tus archivos están ahí.
5
Haz clic en el botón «Save» (Guardar). Está al lado del menú de Branch.
💡 Es un botón gris con letras negras.
⚠️Atención: Si la opción «main» no aparece en el menú, significa que aún no has subido ningún archivo al repositorio. Vuelve al paso 3 y asegúrate de haber subido el index.html correctamente.
🔍 Así debería verse: Después de guardar, la página se recargará y verás un mensaje amarillo que dice: "Your site is being deployed" (Tu sitio se está publicando). También verás una URL parecida a: https://tu-usuario.github.io/mi-primera-pagina/
Settings → Pages
Branch:
main ▼
/ (root)
Save
⏳ Your site is being deployed...
(Tu sitio se está publicando. Espera 2-3 minutos)
⏳¡Ya casi! Apareció el mensaje amarillo. Ahora hay que esperar entre 1 y 3 minutos mientras GitHub prepara tu página. Puedes pasar al siguiente paso mientras tanto.
05

Visitar tu página publicada

¡El gran momento! Ve tu página en Internet y comparte el link.

▾
¡Llegó el momento que esperabas! Si esperaste al menos 2 minutos desde el paso anterior, tu página ya debería estar publicada. Vamos a comprobarlo.

🔽 Encuentra y abre tu página:

1
Vuelve a Settings → Pages (donde estábamos en el paso anterior). O simplemente recarga la página si ya estás ahí.
💡 Recargar = presionar F5 o el icono 🔄 del navegador.
2
Si la página ya está lista, verás un mensaje verde y un link. El link se ve así:
https://tu-usuario.github.io/mi-primera-pagina/
💡 "tu-usuario" es el nombre de usuario que elegiste al registrarte.
3
Haz clic en ese link o cópialo (selecciónalo, Ctrl+C) y pégalo en una nueva pestaña del navegador (Ctrl+V en la barra de direcciones y Enter).
💡 El link se abrirá en una nueva pestaña y deberías ver tu página.
🔍 Así debería verse tu página: Una página blanca o con fondo oscuro (depende del código que usaste) que dice:
¡Hola, mundo!
Esta es mi primera página publicada en Internet.
Hecha por [Tu Nombre] 🚀
⚠️Error común: página no encontrada (404) Si ves una página que dice "404 Not Found", significa que GitHub aún no terminó de publicar. Espera 2 minutos más y recarga la página (F5). Si después de 5 minutos sigue sin funcionar, verifica que tu archivo se llame EXACTAMENTE index.html y que esté en la raíz del repositorio.
🎉¡YA TIENES UNA PÁGINA EN INTERNET! Cualquier persona en el mundo puede verla escribiendo ese link en su navegador. Compártelo con tus amigos, familiares, o por WhatsApp. Pueden abrirlo desde el celular, la tablet o la computadora.
📱Pruébalo ahora mismo en tu celular: Coge tu teléfono, abre WhatsApp, envíate a ti mismo el link, y haz clic en él. Verás tu página aparecer en la pantalla de tu celular. ¡Comparte el link con quien quieras!
06

Cómo actualizar tu página después

Si quieres cambiar algo, editas el archivo y los cambios se publican solos.

▾
Lo mejor de todo: Una vez que activaste GitHub Pages (paso 4), cada vez que subas cambios a tu repositorio, la página se actualiza automáticamente. No necesitas volver a activar nada. Es totalmente automático.

🔽 Para modificar tu página más adelante:

1
Abre tu repositorio en GitHub: github.com/tu-usuario/mi-primera-pagina
2
Haz clic en el archivo que quieras modificar. Por ejemplo: index.html.
3
Se abrirá una vista del contenido del archivo. Arriba a la derecha verás un icono de lápiz ✏️. Haz clic en él.
💡 El lápiz significa "Editar". Si pasas el mouse por encima, dice "Edit this file".
4
El archivo se abrirá en modo edición. Cambia el código directamente en el navegador. Por ejemplo, cambia el texto del título o agrega un nuevo párrafo.
5
Baja con el scroll hasta el final de la página. Verás un recuadro que dice «Commit changes».
6
Escribe un mensaje corto que describa lo que cambiaste. Ejemplo:
Actualicé el título de la página
7
Haz clic en el botón verde «Commit changes».
💡 Hecho esto, GitHub guarda el cambio.
8
Espera 1-2 minutos y recarga (F5) tu página web. Los cambios aparecerán solos.
🔍 Después de editar: Tu página web se actualizará automáticamente. No necesitas hacer nada más. Solo esperar un par de minutos y recargar la página. ¡Así de fácil es mantener tu sitio actualizado!
Editando index.html en GitHub
<h1>¡Hola, mundo!</h1>
<p>Esta página la actualicé yo mismo 🚀</p>
